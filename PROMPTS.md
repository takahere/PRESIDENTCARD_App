# UPSIDER AI Playground - Prompt Templates

このファイルには、非エンジニアの方がAI（Cursor/Claude）に指示を出す際のテンプレートを集めています。
コピー&ペーストして、`[]` の部分を書き換えてご利用ください。

## 1. 新しい画面を作りたい時

```
新しい「[画面名: 例 カード利用明細詳細]」画面を作成してください。
目的は「[目的: 例 特定の決済がなぜ失敗したかを確認する]」ことです。

以下の要件を守ってください：
1. `ui-design.mdc` に従い、iOSライクなレイアウトにする
2. `product.mdc` に従い、エラー表示は攻撃的にならず、解決策を提示するトーンにする
3. データは `mock-data.mdc` を参照し、リアルなデータを入れて表示する
```

### 使用例

```
新しい「通知設定」画面を作成してください。
目的は「ユーザーがプッシュ通知やメール通知のオン/オフを切り替える」ことです。

以下の要件を守ってください：
1. `ui-design.mdc` に従い、iOSライクなレイアウトにする
2. `product.mdc` に従い、設定変更は即時反映され、保存ボタンは不要
3. トグルスイッチを使用し、各設定項目には説明文を添える
```

## Tips

### 良い指示の出し方

1. **目的を明確に**: 「何を」だけでなく「なぜ」も伝える
2. **既存ルールを参照**: `ui-design.mdc` や `product.mdc` を参照するよう指示
3. **具体例を示す**: 曖昧な表現より具体的な例を挙げる

### 避けるべき指示

- ❌ 「いい感じにして」 → 何が「いい感じ」かわからない
- ❌ 「とりあえず作って」 → 目的が不明確
- ❌ 「普通のボタン」 → 既存コンポーネントを使うよう指示すべき

### 推奨する指示

- ✅ 「product.mdcのトーンに従って」
- ✅ 「ui-design.mdcの色を使って」
- ✅ 「mock-data.mdcのリアルなデータで」
